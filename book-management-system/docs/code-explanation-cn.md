# 图书管理系统代码详细解释

本文按项目中的真实文件来解释代码，重点覆盖后端、前端、数据库和配置。  
由于后端主文件较长，解释采用“行号范围加逐行说明”的方式，关键业务代码会逐行讲清楚。

## 一、项目整体结构

```text
book-management-system
├─ pom.xml
├─ run.bat
├─ database
│  ├─ schema.sql
│  └─ data.sql
├─ docs
│  └─ acceptance-guide.md
└─ src
   └─ main
      ├─ java
      │  └─ com/example/library/LibraryApplication.java
      └─ resources
         ├─ application.properties
         ├─ schema.sql
         ├─ data.sql
         └─ static
            ├─ index.html
            ├─ app.js
            └─ style.css
```

- `pom.xml`：Maven 项目配置，声明 Spring Boot、JDBC、H2 数据库依赖。
- `LibraryApplication.java`：后端核心代码，包含启动类、接口、业务逻辑、AI 模块。
- `application.properties`：运行端口、数据库、H2 控制台、AI 参数配置。
- `schema.sql`：建表脚本。
- `data.sql`：初始化账号、分类、图书数据。
- `index.html`：前端页面结构。
- `app.js`：前端交互逻辑和接口调用。
- `style.css`：页面样式。

### 1. 登录

```text
index.html 登录按钮
-> app.js login 方法
-> POST /api/auth/login
-> LibraryApplication.login
-> 查询 users 表
-> 校验密码哈希
-> Session 保存用户
```

### 2. 查询图书

```text
index.html 搜索框和搜索按钮
-> app.js loadBooks 方法
-> GET /api/books
-> LibraryApplication.books
-> 查询 books 和 categories
-> 前端渲染图书卡片
```

### 3. 借书

```text
图书卡片借阅按钮
-> app.js borrowBook 方法
-> POST /api/borrow/{bookId}
-> LibraryApplication.borrow
-> 检查登录
-> 检查库存
-> 库存减 1
-> 新增 borrow_records
```

### 4. 还书

```text
借阅记录归还按钮
-> app.js returnBook 方法
-> POST /api/return/{recordId}
-> LibraryApplication.returnBook
-> 校验用户
-> 状态改为 RETURNED
-> 库存加 1
```

### 5. AI 问答

```text
AI 页面提问按钮
-> app.js askAi 方法
-> POST /api/ai/ask
-> LibraryApplication.ask
-> askLargeModel 尝试真实大模型
-> 失败则 localAiAnswer 本地规则回答
```

## 十、答辩时可以这样讲

本项目是一个基于 Spring Boot 的图书管理系统。前端使用 HTML、CSS、JavaScript 完成页面展示和交互，后端使用 Spring Boot 提供 REST 接口，数据库使用 H2 保存用户、分类、图书和借阅记录。系统实现了登录注册、图书查询、分类管理、借书、还书、管理员统计和 AI 问答等功能。借书和还书接口使用事务保证库存和借阅记录一致，AI 模块支持真实大模型接口，并在接口失败时自动使用本地规则兜底，保证项目在没有网络或没有余额时也能正常演示。
